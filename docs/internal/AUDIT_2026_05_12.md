## 2. Top 5 critiques urgents (≤ 1 semaine, ~10h total)

### 2.1 [SEC-CRIT-01] JS injection via `splash.evaluate_js(f"...")` — CRITICAL
- **File** : `app.py:444-446, 655, 678-688`
- **CWE-95** Eval Injection / **OWASP A03:2021** Injection
- **Probleme** : `splash_window.evaluate_js(f"updateProgress({step}, '{safe_text}', {percent})")` escape manuel `\\` et `'` insuffisant. `\n`, `</script>`, U+2028/U+2029 cassent la string -> execution arbitraire. Token Dashboard transit aussi par la (ligne 678-688).
- **Fix** : `splash.evaluate_js(f"updateProgress({step}, {json.dumps(safe_text)}, {percent})")` partout.
- **Effort** : 30 min. **Confidence** : 0.85.

### 2.2 [SEC-HIGH-04] Token Bearer en `localStorage` — HIGH
- **File** : `app.py:682-686`, `web/dashboard/views/login.js`, `web/dashboard/core/cache.js`
- **CWE-922** Insecure Storage
- **Probleme** : Bearer token ≥32 chars stocke en `localStorage`. Toute XSS exfiltre le token persistent. Le storage `cinesort.dashboard.token` persiste entre sessions.
- **Fix** : cookie HttpOnly + SameSite=Strict + Secure (HTTPS), ou rotation token courte duree. Minimum : ne pas stocker en localStorage persistent (clear sur unload).
- **Effort** : 2h. **Confidence** : 0.85.

### 2.3 [SEC-HIGH-06] SMTP STARTTLS sans cert validation — HIGH
- **File** : `cinesort/app/email_report.py:88-90`
- **CWE-295** Improper Cert Validation
- **Probleme** : `smtp.starttls()` appele sans `context=ssl.create_default_context()`. Default `_create_stdlib_context()` accepte certs auto-signes. MITM LAN intercepte password SMTP.
- **Fix** : `smtp.starttls(context=ssl.create_default_context())`.
- **Effort** : 5 min. **Confidence** : 0.9.

### 2.4 [UI-XSS-MULTI] 4 XSS dans dashboard UI — HIGH
- **Files** : `web/views/film-detail.js:144,157`, `web/components/home-widgets.js:183,191`, `web/components/library-components.js:424`, `web/dashboard/views/library.js:282`, plus 4 `onclick="..."` inline
- **CWE-79** XSS
- **Probleme** : `style="background-image: url('${escapeHtml(poster_url)}')"` — escapeHtml NE FERME PAS l'apostrophe CSS. Et `<img src="${posterUrl}"` sans escape sur dashboard. Plus 4 `onclick="..."` inline qui violent la CSP.
- **Fix** : `safeBgUrl()` helper valide schema URL avant injection. `escapeHtml` sur src dashboard. Migrer onclick inline vers `addEventListener` apres mount.
- **Effort** : 4h. **Confidence** : 0.9.

### 2.5 [REL-HIGH-23] Aucun verrou inter-process — HIGH
- **File** : `cinesort/infra/db/sqlite_store.py` + manquant top-level
- **Probleme** : 2 `CineSort.exe` paralleles sur meme state_dir = moves contradictoires sur meme source. Pas de single-instance lock (mutex Windows nomme ni lock file PID).
- **Fix** : `_get_or_create_lock(state_dir)` au boot, mutex Windows nomme ou lock file avec PID + heartbeat. Refus boot si autre instance active.
- **Effort** : 4h. **Confidence** : 0.85.

## 4. Refactor structurel (3-6 mois)

| Theme | Effort | Quand |
|-------|--------|-------|
| **Casser cycle `domain↔app`** + `import-linter` en CI | 5-10j | **PREMIER** (debloque le reste) |
| **God class CineSortApi** (2187 LOC, 167 methodes) -> facades par bounded context | 8-12j | Mois 1-2 |
| **7 mixins SQLite -> Repositories** (composition au lieu d'heritage) | 5-7j | Mois 3 |
| **Settings v4 vs v5 limbo** : `web/views/settings.js` ET `settings-v5.js` coexistent | 2j | Mois 1 |
| **20 doublons composants v4/v5** (`web/components/*` vs `web/dashboard/components/*`) -> `web/shared/components/` | 5-7j | Mois 2 |
| **50 fonctions > 100 lignes** restantes (Polish v7.7.0 a rate son zero) | 3-5j mecanique | Continu |

### Conseil strategique 6 mois

Adopter un pattern **Hexagonal/Ports & Adapters explicite, avec un `core/` boundary inviolable**.

L'erreur architecturale fondamentale n'est pas le god API ni les mixins — c'est l'**absence de discipline sur la direction des dependances**. Dossiers `domain/app/infra/ui` poses mais sans **garde-fou automatique**. Resultat : `domain/core.py` importe `app/`, `domain/perceptual` importe `infra/`, 162 lazy imports cachent les cycles sous le tapis.

**Plan 6 mois** :
- **Mois 1-2** : Definir dans `cinesort/domain/ports.py` interfaces abstraites (`ProcessRunner`, `MetadataProvider`, `Repository[T]`, `ProbeService`). Refactor `domain/perceptual` pour dependre uniquement de ces ports.
- **Mois 3** : Installer `import-linter` avec contrats stricts. Brancher dans CI. Toute regression layering casse le build.
- **Mois 4-5** : Decouper `CineSortApi` en bounded contexts (RunFacade, SettingsFacade, QualityFacade, IntegrationsFacade).
- **Mois 6** : Migrer mixins en Repositories (`ApplyRepository(connection)`).

Sans cette discipline, dans 1 an : god API de 3000L, 80 fonctions >100L, 250 lazy imports.

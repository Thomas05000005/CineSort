## Par module (detail)

### `cinesort/domain/genre_rules.py:53` — CODE_MORT, severity 1
- **Persona** : QUALITY
- **Description** : La cle `"animation "` (avec espace final) dans `_GENRE_CANONICAL`
  est inaccessible : `canonical_genre()` applique `.strip().lower()` avant lookup,
  donc la cle ligne 45 (`"animation"`) matche systematiquement.
- **Fix** : suppression de la ligne (PR #138, -1L).
- **Impact** : nul (la cle n'etait jamais matchee).

### `cinesort/domain/calibration.py:178-182` — BUG_DOMAINE / RELIABILITY, severity 2
- **Persona** : RELIABILITY / QUALITY
- **Description** : `suggest_weight_adjustment` viole sa docstring ("rééquilibrer
  pour que la somme reste constante") sur poids extremes. Reproduction :
  `{video:89, audio:1, extras:1}` (sum=91), bias underscore focus video :
  apres distribution `{90, 1, 1}` (sum=92), diff=-1, clamp `max(1, 1-1)=1` :
  sum final = 92 ≠ 91.
- **Fix** : repartir le diff iterativement sur les autres categories en respectant
  `[1, 90]` ; retourner `None` si insatisfiable (PR #139, +44/-4L).
- **Impact** : derive silencieuse des poids suggeres en cas extremes (le
  `suggest_*` ne persiste rien automatiquement, l'utilisateur peut refuser).

### `cinesort/domain/title_ambiguity.py:186-196` — CARGO_CULT / CODE_MORT, severity 1
- **Persona** : QUALITY
- **Description** : Try/except cargo-cult autour de `dataclasses.replace()`.
  `replace()` ne leve jamais `TypeError`/`ValueError` sur dataclass valide ;
  le fallback de mutation directe sur frozen dataclass leve `FrozenInstanceError`
  (sous-classe `AttributeError`) capturee silencieusement, donc reset du boost.
  C'est du dead code trompeur.
- **Fix** : simplifier en `promoted = replace(c, score=new_score, note=new_note)`
  (Issue #140, decision humaine).

## Par module

### `cinesort/app/watcher.py`
- **`FolderWatcher.run` ligne 134** — severity **3 (BUG)** — `self._previous_snapshot = current` est execute AVANT le check `_is_scan_running()` (l.137). Sequence problematique :
  - T0 : snapshot=A
  - T1 : snapshot=B (B!=A), scan deja en cours → `continue` mais `_previous_snapshot` est deja remplace par B
  - T2 : snapshot=B (idle) → `_has_changed(B, B)=False` → JAMAIS de scan auto pour le changement A→B
  - **Fix** : decaler `self._previous_snapshot = current` apres le check + apres `_trigger_scan` ; ou marquer un flag `pending_change` qui force le scan au prochain poll.
  - Confidence : 85%.

### `cinesort/infra/_circuit_breaker.py`
- **`CircuitBreaker.call` ligne 92** — severity **2 (QUALITY)** — `except BaseException:` incrémente le compteur `_failures` meme sur `KeyboardInterrupt` (Ctrl+C user), `SystemExit` (shutdown), `GeneratorExit`. Avec un `failure_threshold=10` defaut, 10 Ctrl+C consecutifs pendant des scans peuvent ouvrir le circuit a tort pour 5 min. Remplacer par `except Exception:`. Confidence : 75%.

### `cinesort/infra/rest_server.py`
- **`_serve_dashboard_file` l.464, `_serve_shared_file` l.562, `_serve_views_file` l.614, `_serve_locale_file` l.666** — severity **2 (QUALITY)** — meme squelette repete 4x : resolve relative path, `relative_to(root)` anti-traversal, `read_bytes`, headers Content-Type/Cache-Control/CORS/Request-ID. Seuls les CSP (uniquement sur HTML dashboard) et le prefixe diffèrent. Helper recommande :

  ```python
  def _serve_static_file(self, root, url_path, prefix, *, html_csp=False):
      ...
  ```

  Reduction estimee ~150 LOC. Effort : medium (refactor + tests existant a relancer). Confidence : 90%.
